Worst-case Big-O run time:

createDataSet: O(m) m is the differant types of ages in array
destroyDataSet: O(m) m is the differant types of ages in array 
searchAge: O(m) m is the students in that array of age
insertion: O(1)
deletion: O(n), n is the number of students, which happens if all students have the same age, but usually O(m)
maxAgeGap: O(1)

I used an array of ages called LIST (13 data points) for my data structure, and in each list i had data points AGES with a head node (named STUDENT for conviencence). Each age had a list of STUDENTS which was accessed using a doubly-linked list. This was best best format for my data structure because it allows for searching age to be must faster (sored by age) and can also located the max age gap, makign these two extra function implemented in my code much faster than if it was sorted in any othrr way.
